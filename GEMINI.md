# GEMINI Instructions (JMA Timetable)

## 필수 지침

- 모든 대화는 한국어로 합니다.

## 0. 환경 및 도구 규정
- **Shell**: 모든 명령은 **Windows PowerShell**을 기준으로 작성하고 실행합니다. (예: `&&` 대신 `;` 또는 별도 실행)
- **Git**: 자동으로 커밋이나 푸시를 수행하지 않습니다. 반드시 사용자의 명시적인 지시가 있을 때만 `git commit` 및 `push` 명령을 실행합니다.

## 1. 개발 프로세스 (Core Workflow)

### 0단계: 환경 및 인프라 검증 (Pre-flight Check)
개발 시작 전, 코드 외부의 의존성(네트워크, 환경 변수, 외부 서비스)을 먼저 확인합니다.
- [ ] **환경 변수 존재 확인**: `.env.local`에 필수 키(`NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY`)가 누락되지 않았는지 확인.
- [ ] **DB 연결성 확인**: `npm run db:test`를 실행하여 `DATABASE_URL`이 유효한지 검증. **(반드시 실제 테이블 SELECT 쿼리 결과를 확인해야 함)**
- [ ] **에러 로그 선행 분석**: 발생한 에러 메시지를 통해 인프라 문제인지 로직 문제인지 우선 판단.

### 1단계: 작업 시작 전 체크리스트 작성
모든 명령 수행 전, 수행할 작업의 세부 항목을 체크리스트 형태로 작성합니다.
- [ ] 요구사항 분석 및 이해
- [ ] 테스트 케이스 정의 (TDD)
- [ ] 구현 전략 수립 (필요 시 연결성 테스트 포함)
- [ ] 관련 문서 업데이트 계획

### 2단계: TDD (Test Driven Development) 기반 개발
1. **Red**: 실패하는 테스트 코드를 먼저 작성합니다.
2. **Green**: 테스트를 통과하기 위한 최소한의 코드를 작성합니다.
3. **Refactor**: 코드를 정돈하고 가독성을 높입니다.
*모든 기능은 테스트와 100% 동기화되어야 하며, 테스트가 없는 코드는 승인되지 않습니다.*

### 3단계: 작업 완료 및 검증
명령 종료 전 다음 단계를 수행합니다.
1. **체크리스트 확인**: 시작 시 작성한 체크리스트를 모두 만족했는지 검토합니다.
2. **자동화 도구 실행 (Hooks)**: 모든 도구 실행 후 설정된 `AfterAgent` 훅이 성공하는지 확인합니다.
    - **훅 실패 시 즉시 대응**: 만약 훅 실행 중 에러가 발생하면, 다른 작업을 중단하고 **해당 에러의 원인을 분석하여 즉시 수정**해야 합니다. 에러가 방치된 상태로 다음 단계로 넘어가서는 안 됩니다.
3. **문서 동기화**: 코드 변경 사항이 `docs/` 내 문서에 반영되었는지 확인합니다.

## 2. 코드 품질 및 설계 원칙

- **Clean Code**: 간결하고 가독성이 좋은 코드를 지향합니다.
- **Detailed Comments**: 코드의 의도와 복잡한 로직에 대해 상세한 주석을 작성합니다.
- **Type Safety**: TypeScript의 기능을 최대한 활용하여 런타임 에러를 방지합니다. (Any 사용 지양)
- **Error Learning**: 에러 발생 시 원인을 철저히 분석하고, 동일한 실수가 반복되지 않도록 본 가이드(`GEMINI.md`)에 대응 방안을 즉시 업데이트합니다.

## 3. 에러 대응 및 가이드 개선 히스토리
*(에러 발생 및 해결 시 아래에 원인과 재발 방지책을 기록합니다)*

- **2026-04-21**: 프로젝트 초기 설정 및 GEMINI 가이드 수립.
- **2026-04-21**: DB 연결 시 `ENOTFOUND` 에러 및 Supabase 클라이언트 초기화 에러 발생. 
    - **원인 1**: Supabase 기본 DB 호스트(`db.<ref>.supabase.co`)는 IPv6 전용인 경우가 많아, IPv4 전용 네트워크 환경에서 DNS 해결 실패.
    - **원인 2**: 최신 Supabase는 `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY`를 표준으로 사용하나, 구형 코드 스타일에서 `ANON_KEY`를 참조하여 환경 변수 로드 실패.
    - **재발 방지**: 
        1. **Connection Pooler 사용 강제**: IPv4를 지원하는 Supabase Connection Pooler(포트 6543) 주소 사용을 기본으로 함 (`pgbouncer=true` 옵션 포함).
        2. **표준 명명 규칙 준수**: 최신 가이드에 따라 `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY`를 사용하며, 레거시 `ANON_KEY` 사용을 지양함.
        3. **Pre-flight 검증 강화**: `scripts/test-client.ts`를 실행하여 환경 변수가 실제 코드의 기대치와 일치하는지 항상 먼저 확인.
