# JMA 연습실 예약 시스템 기획안 (Refined V3)

## 1. 서비스 개요

본 시스템은 JMA('준'실용음악학원) 연습실의 예약 및 관리를 디지털화하여 운영 효율성을 높이고 사용자 편의성을 제공하는 것을 목적으로 합니다.

### 서비스 목표

* **실시간성:** Supabase Realtime을 활용하여 다수의 사용자가 동시에 예약 현황을 확인하고 충돌 없이 예약 가능.
* **규정 자동화:** **1회 1시간 예약 원칙** 및 **종료 5분 전 연장 예약** 등 기존 운영 규칙을 시스템적으로 강제.
* **접근성:** 로그인 없이도 예약 가능한 비회원 예약 시스템 및 소셜 로그인 지원.
* **유연한 관리:** 관리자 페이지를 통해 언제든지 연습실 구성을 추가/수정/삭제할 수 있는 동적 관리 환경 제공.
* **모바일 최적화:** 현장에서 스마트폰으로 즉시 예약할 수 있는 반응형 웹 디자인.

---

## 2. 기술 스택 (Technical Stack)

* **Framework:** Next.js 15 (App Router)
* **Language:** TypeScript
* **Database:** PostgreSQL (via Supabase)
* **ORM:** Drizzle ORM
* **Authentication:** Supabase Auth
    * **소셜 로그인:** 카카오(Kakao) 단독 지원
    * **비회원 예약:** 간단한 이름 입력 기반 예약
* **Styling:** Tailwind CSS 4
* **Realtime:** Supabase Realtime (예약 현황 즉시 반영)

## 3. 데이터베이스 모델링 및 데이터 구성

### 연습실 초기 구성 (관리자 변경 가능)

* **1층 (14개):** 1~8번방, 합주실, 드럼 A~D, 미디실
* **2층 (8개):** 1~7번방, 건반레슨실

### `rooms` 테이블

| 컬럼명       | 타입        | 설명                     |
|:----------|:----------|:-----------------------|
| id        | UUID (PK) | 고유 ID                  |
| floor     | INTEGER   | 1 또는 2                 |
| room_name | TEXT      | 1번방, 드럼 A 등            |
| room_type | TEXT      | 일반, 드럼, 미디, 합주실, 레슨실 등 |
| is_active | BOOLEAN   | 운영/비운영 상태              |

### `reservations` 테이블

| 컬럼명        | 타입        | 설명                                           |
|:-----------|:----------|:---------------------------------------------|
| id         | UUID (PK) | 고유 ID                                        |
| user_id    | UUID (FK) | 회원 예약 시 profiles.id (비회원 시 NULL)             |
| guest_name | TEXT      | 비회원 예약 시 입력한 이름                              |
| room_id    | UUID (FK) | 예약된 연습실 ID                                   |
| date       | DATE      | 예약 날짜                                        |
| start_time | INTEGER   | 시작 시간 (예: 13, 14, ...)                       |
| status     | TEXT      | confirmed, cancelled, noshow, lesson(관리자 차단) |

## 4. 핵심 비즈니스 로직

### 예약 정책 (엄격한 1시간 단위 규칙)

1. **단일 시간 예약:** 사용자는 **한 번에 무조건 1시간 분량(1개 슬롯)만 예약**할 수 있습니다. 한 번의 조작으로 2시간 이상의 슬롯을 동시에 예약하는 것은 시스템적으로 불가능합니다.
2. **연장 예약 (종료 5분 전 규칙):** 연속으로 연습실을 사용하고자 할 경우, 미리 예약해 둘 수 없으며 **현재 사용 중인 시간이 끝나기 5분 전(매시 55분)**부터 다음 시간대 예약이 허용됩니다. (
   예: 1시~2시 사용 중이라면 1시 55분이 되어야 2시~3시 타임을 예약 가능)
3. **일일 총량 제한 없음:** 하루 전체 예약 가능 횟수나 총 사용 시간 자체에는 제한이 없습니다.

### 사용자 구분 및 인증

* **회원 (소셜 로그인):** 구글, 네이버, 카카오톡 계정으로 로그인.
* **비회원:** 이름만 입력하여 빠르게 예약.

## 5. UI/UX 및 디자인 시스템

### 메인 화면 (타임라인 그리드)

* **운영 시간 표시:** 이미지 상의 1:00 ~ 9:00+ 시간대 반영.
* **슬롯 상태 구분:**
    * **예약 가능:** 빈 칸 (클릭 시 예약 팝업)
    * **예약 완료:** 예약자 성함 표시
    * **내 예약:** 별도 색상 강조
    * **잠금 상태:** 현재 시간 기준 55분이 되지 않아 아직 연장/예약할 수 없는 인접 슬롯 시각화.

---

## 6. 관리자 기능 (Admin Dashboard)

* **연습실 동적 관리 (Room Management):**
    * 연습실을 새로 추가하거나, 기존 연습실의 이름/층수/분류를 수정할 수 있습니다.
    * 운영하지 않는 방을 활성/비활성(`is_active`) 처리할 수 있습니다.
* **레슨 일정 고정:** 특정 요일/시간대에 반복되는 레슨 슬롯을 일괄 차단 표시.
* **현황판 초기화:** 매일 또는 매주 단위로 전체 예약 현황 관리.
* **사용자 제재:** 허위 예약 또는 노쇼가 반복되는 비회원/회원 차단.
