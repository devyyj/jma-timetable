# JMA Timetable 기술 사양서 (V3)

## 1. 데이터베이스 스키마 (Drizzle ORM)

### `profiles` (회원 정보)
```typescript
export const profiles = pgTable("profiles", {
  id: uuid("id").primaryKey().references(() => authUsers.id),
  fullName: text("full_name").notNull(),
  phone: text("phone"),
  role: text("role", { enum: ["user", "admin"] }).default("user").notNull(),
  provider: text("provider"), // 'google', 'naver', 'kakao'
  createdAt: timestamp("created_at").defaultNow().notNull(),
});
```

### `rooms` (연습실)
관리자 페이지에서 동적으로 CRUD가 가능해야 하므로 `isActive` 및 `sortOrder` 등의 추가 필드 고려.
```typescript
export const rooms = pgTable("rooms", {
  id: uuid("id").primaryKey().defaultRandom(),
  floor: integer("floor").notNull(), 
  roomName: text("room_name").notNull(),
  roomType: text("room_type").notNull(), // 'general', 'drum', 'midi', 'ensemble', 'lesson'
  isActive: boolean("is_active").default(true).notNull(), // 관리자가 비활성화 가능
  sortOrder: integer("sort_order").default(0).notNull(), // 화면 표시 순서
});
```

### `reservations` (예약)
```typescript
export const reservations = pgTable("reservations", {
  id: uuid("id").primaryKey().defaultRandom(),
  userId: uuid("user_id").references(() => profiles.id), // 회원일 경우
  guestName: text("guest_name"), // 비회원일 경우 실명 입력
  guestPhone: text("guest_phone"), // 비회원 연락처 (선택)
  roomId: uuid("room_id").references(() => rooms.id).notNull(),
  date: date("date").notNull(),
  startTime: integer("start_time").notNull(), // 13, 14, ... (1시간 단위이므로 start_time만 저장하면 충분함)
  endTime: integer("end_time").notNull(), // startTime + 1
  status: text("status", { enum: ["confirmed", "cancelled", "noshow", "lesson"] }).default("confirmed").notNull(),
  createdAt: timestamp("created_at").defaultNow().notNull(),
});
```

## 2. 핵심 예약 로직 (Server Actions)

### `createReservation` 프로세스 (1시간 원칙 & 55분 연장 규칙 적용)
1. **입력값 검증:** `roomId`, `date`, `startTime` 유효성 체크. 단일 슬롯(1시간) 요청만 처리.
2. **중복 예약 확인:** 
   ```sql
   SELECT count(*) FROM reservations 
   WHERE room_id = ? AND date = ? AND start_time = ? AND status IN ('confirmed', 'lesson')
   ```
3. **연속 시간 검증 및 55분 룰 적용:**
   * 사용자가 현재 `T` 시간에 해당 연습실을 사용 중이라고 가정했을 때, `T+1` 시간을 예약하려 한다면:
   * **서버의 현재 시간(현재 시각의 '분') 검증:** `T` 시간대의 55분 이후인지(즉, 종료 5분 전인지) 확인.
   * `T` 시간대의 55분 이전이라면 "다음 시간 예약은 현재 사용 중인 시간 종료 5분 전(55분)부터 가능합니다." 에러 반환.
4. **일일 총량 제한 없음:** `userId`나 `guestName`을 기준으로 전체 슬롯 개수를 제한하는 로직은 제외.

## 3. 인증 설정 (Supabase Auth)

### 소셜 로그인 구성
- **Google:** OAuth 클라이언트 ID/Secret 설정.
- **Naver:** Callback URL 등록 및 Client ID/Secret 설정.
- **Kakao:** REST API 키 및 Redirect URI 설정.

### 권한 제어 (RLS Policy)
- `reservations` 조회: 모든 사용자(비인증 포함) 가능.
- `reservations` 생성: 모든 사용자 가능.
- `reservations` 수정/삭제: 
    - 회원은 `userId`가 일치할 때만 가능.
    - 비회원은 세션/쿠키 등을 통한 `temp_token` 일치 시에만 가능.
- `rooms` 테이블: 
    - 조회: 모든 사용자 가능.
    - 생성/수정/삭제: **`role`이 'admin'인 사용자만 가능.**

---

## 4. 실시간 상태 동기화
- `reservations` 및 `rooms` 테이블에 `INSERT`, `UPDATE`, `DELETE` 발생 시 `supabase.channel`을 통해 클라이언트에 브로드캐스팅.
- 특히 관리자가 방의 상태나 순서를 변경했을 때 클라이언트 UI가 즉각 새로고침되도록 적용.
